# Ideas and briefs

Ideas are scored deterministically from durability, product relevance, evidence feasibility, opportunity, effort, and risk. `npm run idea:score` validates the stored total; it does not select or publish an idea.

An approved brief binds one idea to one intended article ID. It defines the research question, user intent, scope, exclusions, required and prohibited claims, source types, testing needs, media needs, product needs, safety concerns, unresolved questions, and editorial recommendation.

Do not create a package first and retrofit governance later. The compiler requires the published idea and approved brief to resolve before an article can enter canonical content.
