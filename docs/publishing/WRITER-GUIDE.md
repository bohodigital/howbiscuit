# Writer guide

Write the answer first, then evidence and presentation. Keep claims within the approved brief. Cite registered sources with `[@source-id]`; do not paste merchant links, prices, rankings, availability, or first-hand claims into prose unless their governed records and evidence are declared.

Writers may edit the package manifest, article Markdown, and package media. Writers must not edit generated schemas, normalized JSON, generated MDX, route registries, or release artifacts.

Use one canonical category and topic for a guide. An editorial standard is categoryless. Use clear headings, descriptive link text, meaningful alt text, and a direct answer that works without decorative presentation. Controlled presentation blocks enhance the page but cannot carry executable MDX.

Before handoff, run `article:validate`. An editor then checks sources, testing state, rights, taxonomy, commerce claims, related articles, and workflow. Publication requires a fresh owner approval for the exact current digest.
